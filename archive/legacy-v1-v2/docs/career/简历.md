**智能电商推荐系统（V1/V2 Multi-Agent）**　　　　　　　　　**AI Agent / 后端开发**　　　　　　　　　**2026.04 - 至今**

**背景**：面向电商个性化推荐场景，将 V1 单次推荐 API 演进为 V2 中心化 Agent Runtime；围绕“导购优先聊天、会话记忆收集偏好、后台画像投影、首页推荐快照、行为反馈闭环”打通推荐系统，并保留 V1 推荐接口兼容能力。

**技术栈**：FastAPI、Python asyncio、Pydantic、SQLite、pytest / pytest-asyncio、OpenAI 兼容接口

**过程**：
1. 设计 `ShoppingManager + Shared Runtime` 分层架构，抽象 `Manager / Worker / Tool / Task / Event / Snapshot` 核心模型，明确用户只与主 agent 交互、固定 worker 池互不通信、tool 统一注册调用的中心化边界，使系统从固定推荐管线演进为可扩展的 Multi-Agent 平台。

2. 实现导购优先聊天链路与规则驱动 planner：支持 `clarify / fallback / advisory / recommendation / comparison` 五类意图分流，默认消息只回答问题和收集偏好，只有显式推荐/比较请求时才返回 `products / comparisons / copies`，把实时链路从同步推荐构建中解耦出来。

3. 实现后台画像投影与纯快照推荐：对话同步只写 `session memory`，偏好稳定后入队 `profile_projection` 事件，由后台生成 `UserProfile` 并继续触发 `homepage` 的 `recommendation_refresh`；`/recommendations` 改为纯快照读取，miss/expired 仅返回空值或旧值并异步入队刷新。

4. 建立 deterministic validation、smoke、background 与 observability 体系，补齐 `bootstrap + pytest + smoke_test_v2` 验证入口，在 API 边界增加结构化日志、trace id、task trace 与 projection trace，保证会话、投影、快照、feedback 全链路可验证、可演示、可排障。

**结果**：V2 轨道 `12/12` 个 feature 全部完成；V2 核心测试与 `python smoke_test_v2.py` 已覆盖会话创建、planner trace、`profile_projection -> recommendation_refresh` 后台链路、首页快照读取与 feedback 闭环；系统同时保留 `POST /recommend` 与 `POST /api/v1/recommend` 兼容入口，完成从 V1 Supervisor 推荐链路到 V2 Shared Runtime + ShoppingManager 平台的平滑演进。
