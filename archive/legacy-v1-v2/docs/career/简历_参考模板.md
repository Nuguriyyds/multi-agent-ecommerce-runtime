**InfiniteChat 即时通信系统**                      后端开发                             2025.12 - 2026.2

**背景**：面向即时通信与轻社交场景，围绕高频发送、低延迟推送、消息可靠性与在线状态一致性等核心问题，设计可验证、可压测、可恢复的分布式 IM 后端架构。

**技术栈**：Spring Boot、Netty、RocketMQ、Redis / Redisson、MySQL、Nacos、OpenFeign、JWT

**过程**：
1.	构建 Netty WebSocket 接入层，采用三层线程模型，实现单设备在线与跨节点精准推送；节点重启后通过指数退避 + 随机抖动重连，将恢复阶段失败重试从 2000 次降至 126 次。

2.	实现消息主链路幂等与会话级有序语义，保证“至少一次投递 + 消费端去重”；3 轮压测与调优，定位多个瓶颈，将发送入口从 491 RPS / P99 48s 优化至 994 RPS / P99 80ms。

3.	设计离线消息热冷分层存储，推送失败消息回流，支持增量补拉与未读计数维护。

4.	实现红包事务链路，发红包通过 MySQL 行锁 / CAS 完成余额扣减，抢红包使用 Lua 原子脚本 + RocketMQ 事务消息；红包接口在 800 RPS 下 P99 51ms零超发、零重复领取。

**结果**：在 2C8G 单节点环境下，RTC 1000 长连接稳定在线，节点重启后约 8s 恢复至 100；消息发送入口在 1000 RPS 目标压测下实测 994 RPS、P99 80ms；红包接口在 800 RPS 下 P99 51ms。
